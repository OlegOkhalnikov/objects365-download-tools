# Почему потребовались сложные скрипты для загрузки Objects365

## 🌏 Проблема: загрузка с китайских серверов

### 1. **SSL/TLS Handshake проблемы**

- **Географическая удаленность**: серверы в Китае (Beijing)
- **Нестабильное SSL-соединение**: частые сбои при handshake
- **Прерывания**: соединение может оборваться на любом этапе
- **Длительные таймауты**: стандартные утилиты "висят" минутами

### 2. **Специфика китайской инфраструктуры**
- **Great Firewall**: нестабильность международных соединений
- **CDN особенности**: ks3-cn-beijing (KS3 - Kingsoft Cloud)
- **Ограничения**: не все методы загрузки работают стабильно
- **SSL-сертификаты**: проблемы с цепочками доверия

## 🔧 Почему одного метода закачки недостаточно

| Метод | Проблема | Решение в скриптах |
|-------|----------|-------------------|
| **HTTPS** | SSL Handshake fails | `--check-certificate=false` |
| **HTTP** | Redirects to HTTPS | Принудительное использование HTTP |
| **aria2c** | Зависает | `timeout 300` + `pkill` |
| **wget/curl** | Обрывы | `--retry` + `--continue` |

## 🎯 Ключевые сложности

### 1. **Размер файлов**
- Каждый архив ~4-8GB
- Общий объем ~327GB
- При обрыве нужно докачивать, а не начинать заново

### 2. **Нестабильность соединения**
```bash
# Типичные ошибки:
- gnutls_handshake() failed
- SSL read: error:00000000:lib(0):func(0):reason(0)
- Connection reset by peer
- Operation timed out
```

### 3. **Разные методы ведут себя по-разному**
- **aria2c**: лучшая докачка, но частые SSL ошибки
- **wget**: стабильнее, но медленнее
- **curl**: хорош для HTTP, проблемы с HTTPS

## 💡 Как работают скрипты

### `download_objects365_full.sh`
- **Агрессивные таймауты**: чтобы не висеть часами
- **5 методов подряд**: если один упал - другой подхватит
- **Принудительный сброс**: `pkill` зависших процессов
- **HTTP fallback**: при проблемах с HTTPS

### `check_targz.sh`
- **Проверка целостности**: битые архивы после обрывов
- **Размер >3GB**: минимальный порог для полного файла

### `redownload_bad.sh`
- **Только проблемные**: не качать всё заново
- **3 метода последовательно**: максимум шансов

## 📊 Статистика типичных проблем
```
Из 51 архива типичная ситуация:
- ✅ 30-35 загружаются с 1-й попытки (HTTPS)
- ⚠️ 10-15 требуют HTTP или другого метода
- ❌ 5-10 остаются битыми/недокачанными
- 🔄 Нужна повторная проверка и докачка
```

## 🎯 Итог
Скрипты решают **три основные проблемы**:
1. **Нестабильность соединения** с китайскими серверами
2. **SSL/TLS ошибки** при handshake
3. **Огромный объем данных**, требующий докачки
